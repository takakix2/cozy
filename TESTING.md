# cozy テストガイド

## 自動テスト

### ビルドテスト
```bash
cargo build --release
```

### ユニットテスト
```bash
cargo test
```

### 機能テストスクリプト
```bash
bash scripts/test_features.sh
```

## 手動テスト手順

### 1. 基本動作確認

#### 起動テスト
```bash
# 新規ファイルで起動
cargo run --release

# 既存ファイルで起動
cargo run --release test.txt
```

#### キーバインドテスト
- `Ctrl+O`: ファイルを開く
- `Ctrl+S`: ファイルを保存
- `Ctrl+F`: 検索モード
- `Ctrl+R`: 置換モード
- `Ctrl+H`: ヘルプ表示
- `Ctrl+G`: Glide（モーダル）モード
- `Ctrl+X`: 終了（変更があれば保存確認）
- `Ctrl+Q`: 保存せず即終了

> キーバインドの正は `src/shortcuts.rs`。上書きは `config.toml` の `[keys]`。

### 2. ペースト / Glide モードテスト

#### クリップボードからのペースト
1. 別アプリでテキストをコピー（システムクリップボードに入れる）
2. エディタ上で `Ctrl+V` を押す
3. クリップボードの内容がカーソル位置に挿入されることを確認

> 専用の Copy/Cut アクションは持たない。行の削除は `Ctrl+K`（Cut Line）、
> コピーは端末/システム側のクリップボード操作に委ねる設計。

#### Glide モード（Vim 風モーダル）
1. `Ctrl+G` で Glide モードに入る
2. `h` `j` `k` `l` でカーソルが移動することを確認
3. `w` / `b` で単語単位、`0` / `$` で行頭・行末へ移動することを確認
4. `gg` / `G` でファイル先頭・末尾へ移動することを確認
5. `dd` で行削除、`x` で1文字削除されることを確認
6. `i` / `a` / `o` で挿入系に入り、Normal に戻れることを確認
7. `3j` のように数字を前置すると回数分繰り返されることを確認
8. `Esc` で Normal モードに戻ることを確認

### 3. シンタックスハイライトテスト

#### Rustファイル
```bash
# Rustファイルを作成
echo 'fn main() {
    let x = 42;
    println!("Hello, world!");
}' > test.rs

# Rustファイルを開く
cargo run --release test.rs
```

確認項目:
- `fn`, `let` などのキーワードがマゼンタ色で表示される
- 文字列 `"Hello, world!"` が緑色で表示される
- コメント `// ...` がグレーで表示される
- 数値 `42` が黄色で表示される

#### Pythonファイル
```bash
# Pythonファイルを作成
echo 'def hello():
    return "world"
' > test.py

cargo run --release test.py
```

#### JSONファイル
```bash
# JSONファイルを作成
echo '{
    "name": "cozy",
    "version": "0.1.0"
}' > test.json

cargo run --release test.json
```

### 4. 選択範囲の可視化テスト

1. エディタを起動
2. 複数行のテキストを入力
3. `Shift+↓` で複数行を選択
4. 選択範囲全体が青背景で反転表示されることを確認
5. カーソル位置が選択範囲内で正しく表示されることを確認

### 5. ペースト時の選択削除テスト

1. テキストを選択
2. 新しいテキストを入力開始
3. 選択範囲が自動的に削除されることを確認
4. 新しいテキストが挿入されることを確認

## 既知の制限事項

- **対話的テスト**: TUIアプリケーションのため、自動化された対話的テストは困難です
- **クリップボード**: Linux環境でのみテスト済み（Windows/Macでは動作が異なる可能性があります）
- **シンタックスハイライト**: 正規表現ベースのため、複雑な構文では不完全な場合があります

## トラブルシューティング

### ビルドエラー
```bash
# クリーンビルド
cargo clean
cargo build --release
```

### クリップボードが動作しない
- Linux環境で `xclip` または `xsel` がインストールされていることを確認
- Wayland環境の場合は、`arboard`が自動的に適切なバックエンドを選択します

### シンタックスハイライトが表示されない
- ファイル拡張子が正しいことを確認（`.rs`, `.py`, `.js`, `.json`, `.toml`）
- ファイル名に拡張子がない場合は、ハイライトされません
